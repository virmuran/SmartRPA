# SmartRPA 节点速查表

> 18 种节点，按六大类组织。不知道怎么选？从「我要对屏幕做什么」出发，找到对应类别即可。

---

## 🫸 鼠标操作

控制鼠标的点击、移动、拖拽滑动。

### 👆 点击 `click`

在屏幕上某个位置或图片上点击一下。

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `template` | 目标模板图片路径 | `"btn_sign.png"` |
| `threshold` | 匹配精度 0-1，越高越严格 | `0.8` |
| `x` / `y` | 直接指定坐标（无 template 时） | `x: 500, y: 300` |
| `roi` | 限定搜索区域 `[x, y, w, h]` | `[0, 0, 800, 600]` |
| `retry` | 失败重试次数 | `3` |
| `verify` | 点击后验证：`{"appear": "tpl.png"}` 验证弹出，`{"disappear": true}` 验证消失 | — |
| `scroll_search` | 自动滚动查找：`{"direction":"down", "amount":300, "max_attempts":5}` | — |

**典型场景：** 点击签到按钮、确认弹窗、列表项。

```
点击"签到按钮"(click) → 等结果 → 继续
```

> 💡 想看「等待结果出现」这个需求，不是在 click 上加 verify，而是单独加一个 `wait_until` 节点，逻辑更清晰。

---

### ➤ 移动 `move`

把鼠标移动到某个位置或图片上，不点击。常用于「悬停触发下拉菜单」。

| 参数 | 说明 |
| --- | --- |
| `template` | 目标模板图片路径 |
| `threshold` | 匹配精度 |

**典型场景：** 悬停"更多"按钮展开下拉菜单、将鼠标移到输入框准备打字。

```
悬停"更多"(move) → 等待下拉展开(wait_until) → 点击目标(click)
```

---

### ↔ 滑动 `swipe`

从 A 点拖拽到 B 点，模拟鼠标拖拽。

| 参数 | 说明 |
| --- | --- |
| `from` | 起点坐标 `[x, y]` |
| `to` | 终点坐标 `[x, y]` |

**典型场景：** 滑动验证码、拖拽排序、滑动解锁。

---

## ⌨️ 键盘操作

模拟键盘输入。

### ⌨ 按键 `press`

按下并释放一个键。

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `key` | 键名（小写字母或特殊键名） | `"enter"`, `"tab"`, `"esc"` |

**典型场景：** 回车确认、Tab 切换焦点、Esc 关闭弹窗。

---

### Aa 输入 `type`

输入一段文字。

| 参数 | 说明 |
| --- | --- |
| `text` | 要输入的文字内容 |

**典型场景：** 在搜索框输入关键字、在表单填写信息。

```
点击搜索框(click) → 输入关键词(type: "Python教程") → 回车确认(press: "enter")
```

---

### 🔑 组合键 `hotkey`

同时按下多个键（如 Ctrl+C、Alt+F4）。

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `keys` | 按键列表，按顺序按下、逆序释放 | `["ctrl", "c"]` |

**典型场景：** 全选(Ctrl+A)、复制(Ctrl+C)、粘贴(Ctrl+V)、关闭窗口(Alt+F4)。

---

## ⏳ 等待

控制流程的时间节奏。

### 🕐 等待 `wait` <span style="color:#9ca3af;">(已隐藏)</span>

> ⚠️ 此节点已从可视化面板隐藏。底层仍支持，但不推荐使用。

暂停指定秒数后继续。

| 参数 | 说明 |
| --- | --- |
| `seconds` | 等待秒数 |

**为什么不推荐：** 固定延迟在加载时间不稳定的场景下不可靠。建议使用 `wait_until`。

---

### 👁 等到 `wait_until`

**持续截屏检查，直到目标画面出现，或超时失败。**

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `template` | 要等待出现的模板图片 | 必填 |
| `timeout` | 最长等待秒数 | `60` |
| `threshold` | 匹配精度 | `0.8` |
| `interval` | 检查间隔秒数 | `1` |

**典型场景：** 点击后页面加载时间不确定——用 `wait_until` 等加载完成的标志出现。

```
点击"提交"(click) → 等到"提交成功"提示出现(wait_until: template="success.png", timeout=30)
```

> 这是 `wait` 的智能替代品。画面一出现立刻继续，不浪费任何一秒。同时自动处理弹窗（在等待期间持续检测）。

## 👁️ 视觉识别

用"看屏幕"来获取信息。

### 🔍 查找 `find`

检查当前屏幕是否包含某个模板图片。**本身不操作鼠标键盘，只是判断「有没有」。**

| 参数 | 说明 |
| --- | --- |
| `template` | 模板图片路径 |
| `threshold` | 匹配精度 |

返回值用作条件判断（SUCCESS = 找到了，FAILURE = 没找到）。

**典型场景：** 放在 `selector` 中作为分支条件；先 `find` 确认存在再执行操作。

```
selector("弹窗处理")
  ├── find("弹窗关闭按钮.png") → click("关闭")  ← 有关闭按钮就点
  └── 直接继续                                ← 没有就算了
```

---

### 🔤 识字 `find_text`

识别屏幕上的文字，检查是否包含某关键词。

| 参数 | 说明 |
| --- | --- |
| `keyword` | 要搜索的关键词 |
| `roi` | 限定识别区域 `[x, y, w, h]` |
| `lang` | OCR 语言，默认 `"chi_sim+eng"` |

**典型场景：** 检查页面上是否出现了「验证成功」「余额不足」等提示文字。

---

### 🎨 颜色 `find_color`

检测屏幕上是否存在指定的颜色区域。

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `target` | 目标颜色 RGB 值 | `[255, 0, 0]` 红色 |
| `tolerance` | 颜色容差，越大越宽松 | `40` |
| `min_pct` | 最小占比 | `0.15`（至少占搜索区域的 15%） |
| `roi` | 限定搜索区域 | — |

**典型场景：** 检测进度条是否满（绿色）、检测错误提示（红色区域出现）。

## 🔀 流程控制

控制节点的执行顺序、分支、重试和循环。

### >> 顺序 `sequence`

**按顺序逐个执行子节点。一个失败，全部停止。**

这是最常用的流程节点，相当于「先做 A，再做 B，再做 C」。

```
sequence("签到流程")
  ├── click("签到按钮")
  ├── wait_until("结果页面")
  └── log("签到完成")
```

---

### ?| 选择 `selector`

**按顺序尝试子节点，哪个先成功就用哪个。全部失败才失败。**

相当于「试试 A，不行就试试 B，还不行就试试 C」。

```
selector("关闭弹窗")
  ├── find("关闭按钮.png") → click("关闭")
  ├── find("取消按钮.png") → click("取消")
  └── press("esc")                    ← 最后兜底
```

---

### ↻ 重试 `retry`

**子节点失败时，重试 N 次。**

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `count` | 最大重试次数 | `3` |
| `interval` | 每次重试间隔秒数 | `1.0` |

每次重试前会重新截图并检测弹窗。

```
retry("关闭弹窗", count=5)
  └── click("关闭按钮.png")
```

> 💡 单个 Action 节点也支持内联 `retry` 参数（如 `"retry": 3`），效果等价但语法更简洁。

---

### ⏱ 超时 `timeout`

**子节点执行超过 N 秒则强制失败。**

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `seconds` | 超时秒数 | `30` |

```
timeout("签到限时", seconds=30)
  └── sequence
        ├── click("签到")
        ├── wait_until("结果")
        └── log("完成")
```

> 全局超时可通过 `_meta.global_timeout` 设置，对整个任务生效。

---

### 🔁 循环 `repeat`

**子节点成功后自动重置重新执行，直到子节点失败才停止。**

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `max_iterations` | 最大循环次数 | `1000` |

相当于 `while (子节点成功) { 继续循环 }`。

```
repeat("批量处理", max_iterations=100)
  └── sequence
        ├── find("未处理项.png")      ← 找不到就失败，停止循环
        ├── click("未处理项.png")
        └── log("处理了一个")
```

## ⚙️ 系统操作

执行系统命令、管理变量和输出日志。

### ▶ 命令 `exec`

执行一条系统命令（cmd/shell）。

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `cmd` | 命令字符串 | `"start msedge https://example.com"` |
| `cwd` | 工作目录 | `"."` |
| `wait` | 是否等待命令结束 | `false` |

**典型场景：** 打开浏览器、启动外部程序、调用脚本。

---

### $= 赋值 `set_var`

设置或更新一个变量，支持多种数据来源。

| 参数 | 说明 |
| --- | --- |
| `name` | 变量名 |
| `from` | 来源：`"value"` 直接赋值 / `"ocr"` OCR 结果 / `"count"` 模板匹配个数 |
| `value` | 当 `from=="value"` 时的直接值 |

```
set_var("counter", from="value", value=0)            ← 初始化计数器
set_var("balance", from="ocr", roi=[100,200,300,50]) ← 从屏幕读数字
set_var("item_count", from="count", template="item.png") ← 数有多少个
```

---

### 📋 日志 `log`

在运行日志中输出一条信息，支持变量替换。

| 参数 | 说明 |
| --- | --- |
| `msg` | 日志内容，用 `{变量名}` 引用变量 |

```
log("当前余额: {balance} 元")
log("共处理了 {counter} 个项目")
```

## 选节点决策树

不知道用什么节点时，按这个路径走：

```
你想做什么？
│
├─ 操作屏幕
│   ├─ 点击 → click
│   ├─ 移动鼠标（不点） → move
│   └─ 拖拽滑动 → swipe
│
├─ 键盘输入
│   ├─ 按一个键 → press
│   ├─ 打字 → type
│   └─ 组合键 → hotkey
│
├─ 等待
│   ├─ 固定时间 → wait
│   └─ 等到画面出现 → wait_until
│
├─ 判断/检查
│   ├─ 图上有没有某个东西 → find
│   ├─ 文字有没有某个词 → find_text
│   └─ 画面有没有某个颜色 → find_color
│
├─ 控制顺序
│   ├─ 按顺序做 → sequence
│   ├─ 多个方案选一个 → selector
│   ├─ 失败多试几次 → retry
│   ├─ 限时完成 → timeout
│   └─ 一直重复做 → repeat
│
└─ 其他
    ├─ 运行命令行 → exec
    ├─ 记录变量 → set_var
    └─ 打日志 → log
```

## 常用组合模式

### 模式 1：稳定点击

```
retry("点击目标", count=3)
  └── click("目标按钮.png", verify={"appear": "成功页面.png"})
```

### 模式 2：弹窗容错

```
selector("可能弹窗")
  ├── sequence
  │     ├── find("弹窗关闭.png")
  │     └── click("弹窗关闭.png")
  └── 什么也不做（自动继续）
```

### 模式 3：列表遍历

```
repeat("逐条处理")
  └── sequence
        ├── find("列表项.png")
        ├── click("列表项.png")
        ├── wait_until("详情加载.png")
        └── log("处理了一条")
```

### 模式 4：带超时的签到

```
timeout("签到", seconds=60)
  └── sequence
        ├── click("签到按钮.png", retry=3)
        ├── wait_until("签到成功.png", timeout=30)
        └── log("签到完成")
```
